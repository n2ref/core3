# Навык: Генерация кода на PHP

## 1. Базовые настройки генерации php
- Генерация кода на PHP 8.2+
- Использовать типизированные параметры и возвращаемые типы (`: type`).
- Избегать устаревших функций (mysql_, ereg и т.д.).
- Для работы с БД использовать PDO или безопасные ORM (не голый mysqli с конкатенацией).
- Всегда экранировать вывод через `htmlspecialchars()` или шаблонизатор.
- Не использовать `eval()`, `create_function()`, `extract($_REQUEST)`.
- Кодировка: `UTF-8` без BOM.
- Окончания строк: `CRLF` (Windows-стиль).
- Отступы: пробелы, размер отступа — `4` пробела, ширина табуляции — `4`.
- Максимальная длина строки: `120` символов.
- Вставка конечной новой строки: отключена.

## 2. Стиль для PHP-файлов

### 2.1. Отступы и переносы

- **Продолжение отступа**: `4` пробела.
- **Отступы в `switch`**: `case` отступает от `switch`, `break` отступает от `case`.
- **Пустые строки**:
    - В коде: максимум `2` пустые строки подряд для одного смыслового блока и 3 строки для разных.
    - В объявлениях (класс, методы): максимум `2` пустые строки.
    - После открывающей `{` и перед закрывающей `}`: разрешено до `1` пустой строки.
    - Вокруг класса: `1` пустая строка.
    - Вокруг метода: `1` пустая строка.
    - После импортов (use): `1` пустая строка.
    - Перед оператором `return`: `1` пустая строка.

### 2.2. Фигурные скобки и расположение

- **Стиль скобок для класса**: на той же строке.
- **Стиль скобок для метода**: на той же строке.
- **Стиль скобок для анонимных функций, блоков (`if`, `for` и т.д.)**: на той же строке.
- **Скобки пространств имён**: стиль `1` (т.е. `namespace Foo;` без скобок — принято в PHP).

### 2.3. Пробелы и операторы

- **Вокруг бинарных операторов**: пробелы (все типы: `+`, `-`, `*`, `/`, `=`, `==`, `&&`, `|`, `&`, `^`, `<<`, `>>`, `??` и т.д.).
- **Вокруг оператора конкатенации (`.`)**: пробелы.
- **После запятой**: пробел.
- **После двоеточия** (в типах возврата, аргументах): пробел.
- **Перед двоеточием** (в типах возврата): пробел.
- **В тернарном операторе**: пробелы вокруг `?` и `:` (но между `?` и `:` нет дополнительного пробела).
- **Вокруг стрелки `->`**: пробелов нет.
- **Приведение типов**: пробел после приведения (`(int) $x`) — отсутствует.
- **Отсутствие пробелов**:
    - Перед запятой.
    - Перед точкой с запятой в `for`.
    - Внутри скобок вызовов методов, функций, `if`, `for`, `catch` и т.д.

### 2.4. Переносы строк

- **Параметры метода** (объявление): перенос отключён, если не помещаются — переносятся.
- **Аргументы вызова**: перенос отключён.
- **Цепочки вызовов** (`->method()->method()`): перенос отключён, но переносятся если длина строки становится больше 120 символов.
- **Бинарные операции**: перенос отключён, знак операции остаётся на той же строке.
- **Тернарные операции**: перенос отключён, если совокупная длина строки меньше 120 символов, иначе переносятся перед знаками `?` и `:`.
- **Список `extends`**: перенос отключён.
- **Атрибуты** (PHP 8+): переносятся построчно.
- **Group use** (объединённые импорты): переносятся каждый элемент на новой строке.

### 2.5. Запятые в многострочных списках

- **После последнего аргумента** в вызове функции/метода: добавлять только если список многострочный.
- **После последнего элемента массива**: не добавлять.
- **После последней переменной в `use` замыкания**: добавлять только если многострочный.
- **После последнего `match`-арма**: не добавлять.

### 2.6. Выравнивание

- **Выравнивание** :
    - Присваиваний.
    - Констант класса.
    - Параметров в многострочных вызовах.
    - Комментариев справа от кода.
    - Пар ключ-значение в массивах.
    - Аргументов в `match`.
    - PHPDoc-комментариев и имён параметров.

### 2.7. Массивы

- **Короткий синтаксис**: `[]`
- **Многострочные массивы**: открывающая скобка на строке, элементы с новой строки, закрывающая скобка на новой строке (по умолчанию). При этом перенос элементов — `off`, IDE сама решит при превышении длины строки.

### 2.8. Комментарии

- **Строковые комментарии** (`//`): начинаются с первого столбца, добавляется пробел после `//`.
- **Блочные комментарии** (`/* */`): остаются как есть.
- **PHPDoc**:
    - Не использовать FQCN внутри тегов.
    - Не оборачивать длинные строки.
    - Не добавлять пустую строку перед тегами.
    - Параметры форматируются с одним пробелом между частями.

### 2.9. Ключевые слова и регистр

- **Ключевые слова** (`if`, `for`, `foreach`, `as`, `return` и т.д.) — **нижний регистр**.
- **Булевы константы** (`true`/`false`) и `null` — **нижний регистр**.

### 2.10. Именование

- **Стиль именования переменных**: `snake_case`, не используйте транслит, бессмысленные сокращения.
- **Свойства**: по умолчанию `private`.
- **Геттеры/сеттеры**: стиль `camel_case` (например, `getUserName`, `setUserName`), порядок — сначала геттеры, потом сеттеры.

### 2.11. Импорты (use)

- **Сортировка**: алфавитная.
- **Перенос group use**: каждый элемент на новой строке.

### 2.12. Разное

- **`declare(strict_types=1)`** — нет принуждения.
- **Пустые классы/методы в одну строку**: запрещено.
- **Оператор `new` со скобками**: писать со скобками.
- **Тип `null` в объединённых типах**: в конце, например `string|null`.


## Итоговые рекомендации

При генерации PHP-кода в среде, использующей данные настройки, модель должна:

- Использовать отступы **4 пробела**, максимальная длина строки **120**.
- Размещать фигурные скобки для **классов и методов на той же строке**,.
- Ставить пробелы **вокруг всех бинарных операторов** и после запятых.
- **Не выравнивать** присваивания, параметры и ключи массивов по вертикали.
- В многострочных конструкциях **добавлять запятую после последнего элемента** только если конструкция перенесена (для аргументов), но **не добавлять** для массивов.
- **Не переносить** короткие цепочки вызовов, параметры и аргументы, если они помещаются в строку.
- Соблюдать **нижний регистр** для ключевых слов PHP.
- Комментарии `//` начинать с первого столбца (без отступа).
- Импорты (`use`) сортировать **алфавитно**.
- PHPDoc оформлять без FQCN (относительные имена классов).
