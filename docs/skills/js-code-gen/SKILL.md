# Навык: Генерация кода на JavaScript

## 1. Базовые настройки генерации кода на JavaScript

- Использовать `const` и `let`, избегать `var`.
- `function` предпочтительнее стрелочных функций для колбэков.
- Всегда `===` и `!==`, не использовать `==`.
- Для работы с DOM избегать `innerHTML` с пользовательскими данными (XSS), использовать `textContent` или `createElement`.
- Для асинхронности предпочтительнее использовать промисы, а не колбэки и `async/await`.
- Кодировка: `UTF-8` без BOM.
- Окончания строк: `CRLF` (Windows-стиль).
- Отступы: пробелы, размер отступа — `4` пробела, ширина табуляции — `4`.
- Максимальная длина строки: `120` символов.
- Вставка конечной новой строки: отключена.

## 2. Стиль для JavaScript-файлов

### 2.1. Отступы и переносы

- **Продолжение отступа**: `4` пробела.
- **Отступы в `switch`**: `case` отступает от `switch`.
- **Отступ для цепочек вызовов**.
- **Пустые строки**:
    - В коде: максимум `2` пустые строки подряд.
    - Вокруг класса: `1` пустая строка.
    - Вокруг метода: `2` пустые строки.
    - После импортов: `1` пустая строка.

### 2.2. Фигурные скобки и расположение

- **Стиль скобок для классов, методов, блоков**: на той же строке.
- **Стиль скобок для `catch`, `finally`, `do-while`**: на той же строке.
- **`else` на новой строке**: нет.
- **Точка в цепочках вызовов** на новой строке: да.

### 2.3. Пробелы и операторы

- **Вокруг бинарных операторов**: пробелы (сложение, вычитание, умножение, деление, присваивание, сравнение, логические, битовые, сдвиги).
- **Вокруг стрелочной функции**: пробел вокруг `=>`.
- **После запятой**: пробел.
- **После двоеточия** в объектах и типах: пробел.
- **Перед двоеточием** в объектах и типах: пробел.
- **В тернарном операторе**: пробелы вокруг `?` и `:`.
- **После унарного отрицания `!`**: ставится пробел.
- **Отсутствие пробелов**:
    - Перед запятой.
    - Внутри скобок вызовов, условий, скобок массивов/объектов.
    - Вокруг оператора `**`.

### 2.4. Переносы строк

- **Параметры метода** (объявление): перенос отключён.
- **Аргументы вызова**: перенос отключён.
- **Цепочки вызовов** (`obj.method().method()`): перенос отключён.
- **Бинарные операции**: перенос отключён, знак операции остаётся на той же строке.
- **Тернарные операции**: перенос отключён.
- **Список `extends`**: перенос отключён.
- **Объектные литералы**: перенос элементов — каждый элемент с новой строки.
- **Массивы**: перенос элементов — каждый элемент с новой строки.
- **Импорты**: каждый импорт на новой строке.

### 2.5. Запятые в многострочных списках

- **Висячая запятая**: Рекомендуется использовать в многострочных списках.

### 2.6. Выравнивание

- Выравнивание:
    - Присваиваний.
    - Параметров в многострочных вызовах.
    - Свойств в объектах.
    - Бинарных операций.
    - Тернарных операций.

### 2.7. Кавычки и точки с запятой

- **Кавычки**: двойные.
- **Точки с запятой**: всегда ставить, но не принудительно.

### 2.8. Комментарии

- **Строковые комментарии** (`//`): добавлять пробел после `//`.
- **Строковые комментарии** с первого столбца: отключено — обычно с отступом.
- **Блочные комментарии** (`/* */`): первый комментарий с первого столбца, пробел после `/*` не добавляется.

### 2.9. Импорты

- **Сортировка**: по алфавиту, по имени модуля — нет.
- **Слияние импортов** из одного модуля: глобально.
- **Предпочтение абсолютных путей**: глобально.
- **Разрешение путей**: через Node.js.
- **Формат импортов** с type: автоматически.

### 2.10. JSX (React)

- **Атрибуты JSX**: значение в фигурных скобках.

### 2.11. Генераторы и стрелочные функции

- **Пробел после `*` в генераторе**: да.
- **Пробел до `*` в генераторе**: нет.
- **Пробел перед асинхронной стрелочной функцией**: да.

### 2.12. Разное

- **`var` / `let` / `const`**: перенос переменных при объявлении — нормальный.
- **Пустые методы/классы в одну строку**: запрещено.
- **Оборачивание комментариев**: отключено.

## 3. Стиль для TypeScript-файлов

- **Отступы**: 4 пробела, continuation indent 4.
- **Пробелы вокруг `:` в типах**: перед двоеточием пробел, после — пробел.
- **Union types**: пробелы внутри, перенос на каждую строку.
- **Enums и const**: перенос на каждую строку.
- **Декораторы классов**: перенос построчно.

## Итоговые рекомендации

- Использовать отступы **4 пробела**, максимальная длина строки **120**.
- Размещать фигурные скобки **на той же строке** для всех конструкций.
- Ставить пробелы **вокруг всех бинарных операторов**, после запятых, после двоеточий.
- **Выравнивать** присваивания, параметры, свойства объектов.
- **Добавлять висячую запятую** в многострочных объектах и массивах (согласно `keep`).
- **Ставить точку с запятой** после каждого выражения.
- Использовать **двойные кавычки**.
- В цепочках вызовов переносить **точку на новую строку**.
- Импорты оформлять **каждый на новой строке**, сортировать алфавитно.
- Комментарии `//` начинать **с пробела после `//`** и с текущего отступа (не с первого столбца).
- **Не переносить** короткие аргументы, параметры, бинарные операции.
