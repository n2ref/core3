# Рекомендации по проектированию API

При проектировании REST API следуйте этим правилам:

## Структура URL

- Используйте существительные во множественном числе для ресурсов: `/users`, `/orders`
- Используйте camel_case для ресурсов, состоящих из нескольких слов: `/order_items`
- Вложенные связанные ресурсы: `/users/{id}/orders`

## Методы HTTP

- GET: Получение ресурсов
- POST: Создание новых ресурсов
- PUT: Замена всего ресурса
- PATCH: Частичное обновление
- DELETE: Удаление ресурса

## Коды ответа

- 200: Успех
- 201: Создано
- 400: Неверный запрос
- 404: Не найдено
- 500: Ошибка сервера
